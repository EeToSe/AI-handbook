# 长对话与项目管理

本页说明如何组织 AI 对话，避免长对话造成上下文混乱、结论丢失或任务边界不清。

## 1. 项目组织

1. 每个研究项目使用独立 Project。
2. 每个子任务新建独立 chat，例如“文献综述”“代码复现”“论文第 3 章”“审稿回复”。
3. 不要把多个项目、论文或代码仓库混在同一个长对话中处理。

## 2. 阶段摘要

长对话进行到阶段性节点时，应要求模型输出状态摘要，包括：

1. 当前目标。
2. 已完成内容。
3. 关键约束。
4. 关键结论。
5. 未解决问题。
6. 下一步任务。

将状态摘要复制到新 chat，再继续推进后续任务，避免在同一对话中无限延长上下文。

## 3. 复杂任务要求

对复杂任务，应明确要求模型先列假设，再给分析过程，最后给可执行结果。涉及论文结论、实验解释、代码修改和项目决策时，应要求模型说明依据和不确定性。

## 4. 结果沉淀

重要结论应同步到 Obsidian、Git 仓库、项目 README、论文 claim 文档或实验记录中，不应只保留在聊天记录里。
