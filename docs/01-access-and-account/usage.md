# 模型与额度使用原则

本页说明课题组成员如何按任务复杂度选择模型，避免高算力模型额度被低价值任务消耗。

## 1. 模型选择

| 任务类型 | 推荐选择 |
|---|---|
| 日常问答、术语解释、简单翻译、普通润色 | Instant 或轻量模型 |
| 文献观点梳理、论文结构设计、复杂推理、代码审查 | Thinking 模型 |
| 长文档审阅、高质量论文推敲、复杂项目规划、多轮代码重构 | 高级 Thinking 模型 |
| 大规模资料检索、需要来源核验的问题 | Deep Research 或带浏览能力的模型 |
| 仓库级代码修改、测试、重构、脚本生成 | Codex 或其他编程代理工具 |

## 2. 使用原则

1. 先用低成本模型澄清问题、整理材料和生成初稿。
2. 再把目标明确、材料完整、价值较高的任务交给高算力模型。
3. 不把高级模型用于闲聊、低难度搜索、格式转换或无明确目标的长对话。
4. 只有在普通 Thinking 模型无法完成任务时，再使用更高级的 Thinking 模型。

## 3. GPT-5.5 Pro 额度与 fallback

Pro 订阅提供的动态使用额度明显高于 Plus，但 GPT-5.5 Pro 这类高算力模型通常仍有单独 allowance。额度用完后，GPT-5.5 Pro 可能临时不可用，也可能在继续提问时 fallback 到较轻量的模型。长文档、复杂代码和深度推理任务会更快消耗 GPT-5.5 Pro 额度。

正常情况下，用户选择 Thinking 模型后，模型输出说明应与所选模型一致。例如选择 GPT-5.5 Thinking 时，输出说明应显示为 GPT-5.5 Thinking。

![正常情况下选择 GPT-5.5 Thinking 后的模型输出说明](assets/GPT-Thinking-normal.png)

当 GPT-5.5 Pro 的单独 allowance 用完后，再次提问可能不会继续使用原本选择的 Pro 模型，而是 fallback 到较轻量的模型，例如 GPT-5.3 mini。
